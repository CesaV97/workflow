Key organizational principles:
• components/ organized by feature domain (Layout, Pages, TaskDetail, Projects, Tasks)
• hooks/ contain shared state logic (projects, tasks, timer, storage)
• data/ holds JSON files for Phase 1 (will be replaced with backend in Phase 2)
• styles/ separated by concern (global, theme, layout, components)
• TaskDetailPanel is a conditional overlay that doesn't change the page
• All data flows through custom hooks to a single source of truth
