✅ Feature-Level Styling Benefits

One file per feature — All Dashboard styles in Dashboard.css
Easy to delete — Remove a feature? Delete the folder + its CSS
Cleaner imports — Components import from one CSS file per feature
Consistent styling — All related styles in one place
Cascading styles — Use CSS nesting for component-specific rules
Team alignment — Clear boundaries, no style conflicts

Structure summary:
• features/ contains all feature-specific components (Dashboard, Projects, Tasks, Pomodoro, Comments, Reports, Settings)
• Each feature has ONE CSS file that contains all styles for all components within that feature (e.g., Dashboard.css covers Dashboard, ProjectGrid, WeeklyTasksList)
• CSS organization uses class naming conventions or nesting to organize styles by component
• Global styles only for theme variables and base resets
• components/ contains shared components (Layout, Common UI) with their own feature-level CSS files
